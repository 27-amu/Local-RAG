Introduction to Retrieval-Augmented Generation (RAG)

Retrieval-Augmented Generation (RAG) is an AI framework that combines the strengths of large language models (LLMs) with external knowledge retrieval. Instead of relying solely on the knowledge baked into the model during training, RAG allows a model to fetch relevant documents from an external knowledge base at inference time and use that context to generate more accurate, up-to-date, and grounded answers.

How RAG Works:
1. Ingestion Phase: Documents are loaded, split into chunks, converted into embeddings (numerical vectors), and stored in a vector database.
2. Retrieval Phase: When a user asks a question, the query is converted into an embedding. The vector database finds the most semantically similar document chunks using cosine similarity or other distance metrics.
3. Generation Phase: The retrieved chunks are passed as context to the LLM along with the original question. The LLM generates an answer grounded in the retrieved documents.

Key Components:
- Document Loader: Reads raw files (PDF, TXT, HTML, DOCX, etc.) and converts them into Document objects.
- Text Splitter: Breaks large documents into smaller chunks (e.g., 750 tokens with 50 token overlap) to fit within the LLM's context window.
- Embedding Model: Converts text into dense vector representations (e.g., all-mpnet-base-v2 from HuggingFace).
- Vector Store: A database optimized for similarity search over embeddings. ChromaDB is a popular choice for local setups.
- LLM: The language model that generates the final answer (e.g., llama3, phi3 via Ollama).
- Retriever: Fetches the top-k most relevant chunks from the vector store for a given query.

Benefits of RAG:
- Reduces hallucinations by grounding answers in real documents.
- Allows use of up-to-date information without retraining the model.
- Works entirely locally for privacy-sensitive applications.
- Cost-effective compared to fine-tuning.

This project uses LangChain to orchestrate the RAG pipeline, ChromaDB as the local vector store, HuggingFace sentence-transformers for embeddings, and Ollama to run the LLM locally on your machine.
